Chantel Boor Bank Project

Walk Through UX

User begins with a prompt to say if they are a member. Mebers are directed to login and non-members to join. On login page program will take input and find the user by their email. If the email is found the user is logged in, if not the program says login is incorrect and prompts to try again. Email and password must be correct in order to login

If a User says they are not a member they will be directed to the new users page. This takes in user info and passes it to service layer. Service layer checks if email is in use then allows the dao to call the database and create a new entry if the email is not taken. User enters all info except primary key. The Primay key is handeled by a trigger and sequence within the database. Once finished user is taken to login screen

Here the service layer is automatically called so it can call the necessary dao methods to display current accounts or the text no accounts if the user has none. From here the user can input one of eight menu options to preform the functions avalible to them. Below a few examples will be disscussed since all menu choices behave in a simular way, by going through service layer and then doa to call appropriate database actions.

Here the user chooses to add a new account. The service layer method that displays classes is called at the beging and printed here for the users. The user inputs their choice of what account type and between the service layer and database triggers the other info is filled in for them. The service layer also checks if the account type is valid and if the user is allowed to have more accounts before allowing the account to be created.

Here the user closes an account by inputing the id of an account displayed above. The service layer will check if the account number exists and if the balance is zero and then call the dao. The dao then calls a prepared statement that change the close date to current date and then removes the account from the join table where the dao searches for accounts that a user has.

Here the user deposits money in an account by inputing one of the account numbers above and the amount. From here the service layer is called which will check if the account exists and fetch an account object. It then sets the balance by added the deposit amount to the current amount. Then the dao is called to update the value in the table with a prepared statement
